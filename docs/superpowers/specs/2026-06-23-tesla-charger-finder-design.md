# 테슬라·전기차 충전소 찾기 웹앱 — PRD (설계 문서)

작성일: 2026-06-23
작성: Claude Code (이현석 디렉터와 브레인스토밍)
상태: 검토 대기 (디자인은 실제 빌드에서 디벨롭)

---

## 1. 한 줄 정의

내 위치(또는 검색한 주소) 주변의 전기차 충전소를 네이버 지도 위에 표시하고, **지금 빈 자리가 몇 개인지(실시간)** 를 한눈에 보여주는 모바일 우선 웹앱. 테슬라 슈퍼차저는 별도로 강조한다.

## 2. 왜 만드는가 (문제)

- 테슬라를 새로 구입해, 여행·이동 중 급속/완속 충전소를 빠르게 찾고 싶다.
- 갔는데 모든 자리가 충전 중이라 허탕치는 일을 줄이고 싶다 → **실시간 빈자리 수**가 핵심.
- 앱을 켜자마자 "내 주변 어디서 충전 가능한지"가 직관적으로 보여야 한다.

## 3. 사용자 & 핵심 시나리오

- 주 사용자: 디렉터 본인(테슬라 차주). 친숙함·편의성이 최우선.
- 시나리오 A: 앱을 켠다 → 위치 권한 허용 → 내 주변 충전소가 지도에 뜬다 → 빈자리 있는 곳(초록)을 보고 누른다 → 네이버 길찾기로 출발.
- 시나리오 B: 여행 목적지를 검색창에 입력 → 그 지역 주변 충전소를 미리 확인.
- 시나리오 C: "테슬라"만 필터 → 슈퍼차저만 본다.

## 4. 핵심 기능 (MVP)

1. **현재 위치 기반 표시**: 위치 권한 허용 시 내 위치 중심으로 주변 충전소 표시. 거부 시 지도만 띄우고 검색 유도.
2. **주소·지명 검색**: 검색해서 그 지역으로 지도 이동 + 주변 충전소 표시.
3. **충전소 마커**: 지도 위에 충전소를 마커로. 마커 색으로 상태 구분(아래 6번 규칙).
4. **실시간 빈자리 현황**: "급속 5대 중 2대 사용가능 · 3대 충전중" + 마지막 갱신 시각.
5. **상세 카드(바텀시트)**: 마커를 누르면 이름/실시간 빈자리/커넥터·속도/운영기관·요금·운영시간/거리/길찾기 버튼.
6. **필터**: 전체 / 급속 / 완속 / 테슬라 / 빈자리만(토글).
7. **테슬라 슈퍼차저 강조**: 별도 색·아이콘.
8. **지도를 움직이면** 그 영역 주변 충전소를 다시 불러옴.
9. **지도 스타일 토글 버튼**: 상세 지도 ↔ 간단한 2D 지도(덜 복잡한 뷰)를 버튼으로 전환.
10. **네이버 길찾기 연동**: 카드의 버튼으로 네이버 지도 길찾기 실행.

## 5. 가능 / 불가능 (정직하게)

- ✅ **가능**: 실시간 "빈 자리 수"(충전기별 상태로 집계), 충전 속도·커넥터 타입, 운영기관/요금/운영시간, 거리, 길찾기 연결.
- ❌ **불가능**: 개별 차량의 "충전 잔여 시간 / 곧 자리 날 듯" — 공개 API에 잔여시간 필드가 없음(마지막 충전 시작·종료 시각만 존재). 이 기능은 만들지 않는다.

## 6. 데이터 소스 (검증 완료)

### 6-1. 전체 충전기 — 한국환경공단 공공 API
- 데이터셋: 한국환경공단_전기자동차 충전소 정보 (data.go.kr/data/15076352)
- 서비스 URL: `http://apis.data.go.kr/B552584/EvCharger`
  - `getChargerInfo` — 충전소·충전기 기본정보(위경도, 충전기타입, 주소, 운영기관, 충전용량 등)
  - `getChargerStatus` — 충전기 실시간 상태(stat, 상태갱신시각, 마지막 충전 시작/종료)
- 제공 필드(핵심): 위도(lat)·경도(lng), 충전기타입(chgerType: 급속/완속·커넥터 구분), 충전용량(output, kW), 운영기관(busiNm), 주소(addr), 상태(stat), 상태갱신시각(statUpdDt), 지역코드(zcode 시도/zscode 시군구).
- 상태코드(stat) 매핑 — 일반적으로 알려진 값(빌드 시 공식 문서로 최종 확정):
  - 1 통신이상, 2 충전대기(=사용가능), 3 충전중, 4 운영중지, 5 점검중, 9 상태미확인
- 호출 제약: 개발계정 1일 1,000건 → **서버 캐싱 필수**. 위치/지역 단위 질의는 `zcode`(시도)로 받고, 반경 필터는 좌표 거리로 처리.

### 6-2. 테슬라 슈퍼차저 강조 — 디렉터의 네이버 즐겨찾기
- 출처: 디렉터가 직접 모은 네이버 지도 즐겨찾기 폴더(테슬라 슈퍼차저).
- 사용법: 해당 목록의 좌표/이름을 별도 데이터로 추출해 "테슬라" 레이어로 강조 표시.
- (대안/보강: 공공 API에서 커넥터타입이 테슬라이거나 운영기관이 테슬라인 충전기도 테슬라로 분류 가능 — 빌드 시 정확도 비교 후 결정.)

## 7. 아키텍처 (A안 확정)

```
[브라우저 / 모바일 웹]
   └ 네이버 지도 JS SDK (지도·마커·검색·길찾기 연동)
   └ 내 위치(Geolocation) / 주소 검색
        │  (충전소 데이터 요청: 위경도 또는 지역)
        ▼
[Vercel 서버리스 함수]  ← 비밀키 숨김 + 캐시(1,000건 제한 대응)
   └ 한국환경공단 공공 API 호출(getChargerInfo + getChargerStatus 결합)
   └ 요청 영역(반경/시도)으로 필터링해서 필요한 만큼만 응답
        │
        ▼
[정적 프론트엔드]  ← Vercel 배포 (지금 쓰는 워크플로 그대로)
```

- 무거운 백엔드/DB 없음(YAGNI). 서버리스 함수 1~2개 + 정적 페이지.
- 캐시 전략(개요): 같은 지역/영역 요청은 짧은 TTL(예: 1~3분)로 캐싱해 호출 수와 응답 속도 모두 개선. 정확한 TTL은 빌드 중 결정.

## 8. 마커 / 상태 표시 규칙

- 🟢 초록: 지금 빈자리 있음(사용가능 대수 ≥ 1)
- 🔴 빨강: 테슬라 슈퍼차저(강조)
- ⚪ 회색: 만차 / 점검 / 운영중지
- 🔵 파란 점: 내 위치
- 마커 안 숫자 = 지금 충전 가능한 대수
- (이모지가 아니라 SVG 아이콘으로 구현 — 실제 앱 느낌 유지)

## 9. 비기능 요구사항

- **모바일 우선**: 여행 중 폰으로 보는 게 기본. 세로 화면, 큰 터치 타깃, 한 손 조작.
- **속도**: 켜자마자 빠르게 주변이 떠야 함(서버 캐시로 보조).
- **친숙함**: 네이버 지도 기반, 한국 사용자에게 익숙한 UI 패턴.
- **보안**: 공공 API 키·네이버 지도 키는 서버/환경변수에만. 프론트에 노출 금지.
- **한글 가독성**: word-break:keep-all, 의미 단위 줄바꿈(프로젝트 공통 규칙).

## 10. 디렉터가 직접 해야 할 일 (둘 다 무료, 단계별 안내 제공)

1. 공공데이터포털에서 "한국환경공단_전기자동차 충전소 정보" 활용신청 → 인증키 발급.
2. 네이버 클라우드 플랫폼(NCP)에서 지도(Maps) 서비스 신청 → 클라이언트 키 발급.
3. 테슬라 슈퍼차저 네이버 즐겨찾기 목록 공유(좌표 추출용).

## 11. 범위 밖 (YAGNI / 향후 후보)

- 충전 잔여시간 예측(데이터 없음 — 불가).
- 로그인/회원/즐겨찾기 저장(MVP 제외, 향후).
- 결제·예약(범위 밖).
- 경로상 충전소 자동 추천(여행 경로 따라) — 향후 후보.
- 안드로이드/iOS 네이티브 앱 — 웹앱으로 시작.

## 12. 빌드 중 확정할 미해결 항목

- stat 상태코드 정확한 매핑(공식 문서 최종 확인).
- chgerType 코드 → 급속/완속·커넥터(테슬라/DC콤보/차데모/AC3상) 매핑표.
- 테슬라 슈퍼차저를 "즐겨찾기 목록"으로 할지 "공공 API 분류"로 할지(정확도 비교 후).
- 지도 이동 시 재조회의 디바운스/캐시 TTL 수치.
- 요금 표기: 공공 API 요금 필드 유무 확인 후, 없으면 "운영기관별 대략" 또는 생략.
- 간단한 2D 지도 모드: 네이버 지도 mapType 전환으로 충분한지, 아니면 별도 경량 스타일이 필요한지.

## 부록 A. 데이터 아키텍처 v2 — 워밍 캐시 (2026-06-23 빌드 중 확정)

빌드 중 실측으로 드러난 사실이 원래 "앱이 매 요청마다 공공 API 직접 호출" 전제를 깼다. 이에 따라 데이터 전달 구조를 아래로 변경한다.

### 실측 (근거)
- 공공 API 응답 시간이 **호출당 30~44초로 고정**(numOfRows 10=44s, 100=30s, 1000=31s) — 크기 무관, 정부 서버 자체 지연.
- 시도(zcode)별 충전기 수가 큼: 서울 75,044 · 경기 152,757 · 부산 32,314 · 제주 8,600. 시군구(zscode)는 작음: 강남구(11680) 5,930(1페이지).
- 따라서 **앱 요청 중 공공 API 동기 호출 = Vercel 타임아웃**. 시군구로 좁혀도 호출 1건이 ~40초라 동일.

### 구조 (확정)
1. **저장소**: Vercel KV. 키 `sigungu:<zscode>` → `{ stations: Station[], updatedAt }`. 앱은 여기서 즉시 읽음.
2. **워밍 워커**(Vercel 밖, **맥미니 cron**, 한국시간 기준 ~10분 간격): KV에 들어 있는(=최근 사용된) 시군구들을 다시 공공 API로 받아 `buildStations`로 가공해 KV에 갱신. 40초 지연은 백그라운드라 UX 무관.
3. **앱 핸들러 `/api/chargers`**: 프론트가 넘긴 시군구 코드(zscode, 네이버 역지오코딩으로 도출)로 KV 조회.
   - 신선한 데이터 있으면 거리 필터 후 즉시 반환.
   - 없으면 `{ status: 'warming' }` 반환 + KV에 "요청됨" 표시(다음 워밍 주기에 채워짐). 프론트는 로딩 표시 후 폴링.
4. **프론트**: 지도 중심 → 네이버 역지오코딩으로 시군구 코드 얻어 호출. warming이면 "충전소 불러오는 중(공공데이터가 느려요)" 표시 후 몇 초 간격 폴링.

### 결과
- 자주 가는(=KV에 살아있는) 시군구: 항상 즉시 + 실시간 ~10분 이내.
- 처음 가는 시군구: 1회 워밍 주기만큼 대기(로딩 UI), 이후 워밍 목록에 들어가 계속 즉시.
- 실시간 "빈자리 수": getChargerInfo의 자체 `stat`으로 집계(별도 status 엔드포인트는 누락이 커서 미사용).

### 인프라 분담
- 이현석 님: ① Vercel 대시보드에서 KV 추가(로그인 필요), ② 맥미니 스케줄 설치 시 승인 + 같은 와이파이(SSH 접속용).
- Claude: store 추상화·워머 스크립트·핸들러·프론트·스케줄 파일 작성, 맥미니 SSH 설치(연결되면), 배포.

### 미해결(빌드 중 확정)
- 좌표→시군구코드(zscode) 변환: 네이버 reverse geocode 응답의 행정구역 코드 매핑.
- 경계 인접 시군구 동시 조회 여부(반경이 구 경계를 넘을 때).
- KV/워머가 쓸 인증(KV_REST_API_URL/TOKEN) 맥미니 배치.
- 맥미니 접속: 같은 네트워크 필요(현재 맥북은 다른 망 → SSH 불가). 막히면 GitHub Actions 폴백.
